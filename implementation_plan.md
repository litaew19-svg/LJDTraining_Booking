# แผนการพัฒนาระบบล็อกอิน ปฏิทิน และการตั้งค่าแพ็กเกจ (Implementation Plan) - [UPDATED & COMPLETED]

แผนการทำงานและภาพรวมสถาปัตยกรรมของระบบจองคลาสเรียน (PT Booking Platform) สำหรับ **LJD Training** ซึ่งได้รับการพัฒนาและทดสอบเสร็จสมบูรณ์เรียบร้อยแล้วตามความต้องการทุกประการ

---

## 📐 รายละเอียดฟีเจอร์และเงื่อนไขทางธุรกิจ (Business Rules & Features)

### 1. หน้าจอลงทะเบียน / เข้าสู่ระบบ (Auth Screen)
- บังคับการล็อกอินด้วยอีเมลหรือเบอร์โทรศัพท์ (ค้นหาจากฐานข้อมูลจำลอง `localStorage` ของระบบ)
- สมาชิกใหม่สามารถลงทะเบียนเป็นบทบาท "Student" 
- ระบบจะสร้างบัญชีผู้ใช้ใหม่และสามารถกดซื้อแพ็กเกจ (Group Class หรือ 1-on-1 PT) เพื่อเพิ่มสิทธิ์เครดิตในการจองคลาสเรียนต่าง ๆ ได้

### 2. การจำกัดสิทธิ์ (Role Access Control) & การจัดการบัญชี
- **ผู้เรียน (Student)**: ใช้งานได้เฉพาะหน้าแดชบอร์ดนักเรียนเท่านั้น แถบสลับบทบาท (Role Switcher) จะถูกซ่อน
- **เทรนเนอร์ (Trainer)**: ใช้งานได้เฉพาะหน้าแดชบอร์ดเทรนเนอร์ แถบสลับบทบาทจะถูกซ่อน
- **แอดมิน (Admin)**: แสดงแถบสลับบทบาท "View As" ด้านบน เพื่ออำนวยความสะดวกในการจำลองและทดสอบการบริหารจัดการระบบในทุกบทบาท
- แสดงชื่อผู้ใช้ปัจจุบันที่ล็อกอินอยู่พร้อมปุ่ม **Logout** ด้านขวาบนของทุกหน้าจอ

### 3. ระบบปฏิทินรายเดือน (Monthly Calendar)
- แสดงปฏิทินแบบ Interactive สำหรับตรวจสอบวันเรียน
- **แถบสีตัวระบุสถานะบนปฏิทิน (Calendar Indicator Colors)**:
  - **ฝั่งนักเรียน (Student)**: จุดกลมและขอบด้านล่างสีเขียวสำหรับวันที่ระบุว่ามีคลาสเรียนเปิดสอนอยู่ (**Available Class**) และจุดกลม/ขอบด้านล่างสีน้ำเงินสำหรับวันที่นักเรียนได้กดจองคลาสเรียนเอาไว้ (**Your Booking**)
  - **ฝั่งครูฝึก (Trainer)**: แสดงจุดสีเขียวเมื่อมีตารางสอนของตนเองในวันนั้น
- ผู้ใช้สามารถคลิกวันที่ในปฏิทินเพื่อกรองแสดงเฉพาะคลาสเรียนของวันนั้น ๆ ในรายการด้านล่างปฏิทินได้ทันที พร้อมปุ่มล้างตัวกรอง (Clear Filter)

### 4. อัตราค่าเรียนและการหักเครดิต (Pricing & Booking Rules)
- **คลาสกลุ่ม (Group Class)**:
  - การจองคลาสกลุ่มจะหักจากแต้มเครดิตคลาสกลุ่ม (**Group Class Credits**) 1 เครดิตต่อครั้ง
  - **เงื่อนไขจำนวนคนขั้นต่ำ**: หากจำนวนคนในคลาสน้อยกว่าที่แอดมินกำหนด (ค่าเริ่มต้นคือ 2 คน) ระบบจะแสดงสถานะคลาสว่า **"待成班 (Pending)"** เมื่อมีผู้เข้าร่วมตั้งแต่ 2 คนขึ้นไป ระบบจะอัปเดตสถานะเป็น **"已成班 (Confirmed)"** ให้โดยอัตโนมัติ
- **คลาสส่วนตัว (1 on 1)**:
  - ซื้อ 1 คลาส: 1,800 NT / คลาส (อายุ 7 วัน)
  - ซื้อ 10 คลาส: 1,450 NT / คลาส (อายุ 90 วัน)
  - ซื้อ 20 คลาส: 1,350 NT / คลาส (อายุ 180 วัน)
  - การจองคลาสส่วนตัวต้องหักจากแต้มเครดิตคลาสส่วนตัว (**1-on-1 Credits**) 1 เครดิตต่อครั้ง
- **กฎการยกเลิกการจอง (Cancellation Policy)**:
  - สามารถกดยกเลิกการจองคลาสได้ก่อนเวลาเริ่มเรียนล่วงหน้าอย่างน้อย **24 ชั่วโมง (1 วัน)** หากเหลือน้อยกว่า 24 ชั่วโมง ระบบจะไม่ยินยอมให้กดยกเลิก
  - เมื่อยกเลิกสำเร็จ ระบบจะคืนเครดิตประเภทนั้น ๆ กลับเข้าสู่แพ็กเกจของนักเรียนโดยอัตโนมัติ

### 5. แผงควบคุมตั้งค่าและระบบจัดการสำหรับแอดมิน (Admin Settings & Class Management)
- **แผงตั้งค่าราคาแพ็กเกจ (Pricing settings)**: แอดมินสามารถปรับแก้ราคาสำหรับทดลองเรียน/เรียนปกติของคลาสกลุ่ม, จำนวนคนขั้นต่ำในการเปิดห้องเรียน, และราคาแพ็กเกจ 1-on-1 PT แบบขั้นบันไดได้โดยตรงผ่านฟอร์มการตั้งค่า
- **การจัดการคลาสเรียน (Session Control)**:
  - แอดมินสามารถกดปุ่ม **"Add Class Session"** เพื่อสร้างคลาสเรียนใหม่ ๆ ได้เองทันที
  - เพิ่มปุ่มแก้ไข (**Edit Class**) และปุ่มลบ (**Delete Class**) ในหน้าต่าง All Sessions
  - **ระบบคืนเครดิตเมื่อลบคลาส**: หากแอดมินกดลบคลาสเรียนที่นักเรียนจองไว้แล้ว ระบบจะทำการลบประวัติการจองและคืนแต้มเครดิตให้กับนักเรียนทั้งหมดที่จองคลาสนั้นทันทีเพื่อความปลอดภัยของข้อมูล
- **ระบบเพิ่มประเภทคลาสเรียนใหม่ ๆ (Custom Class Types)**:
  - แอดมินสามารถสร้างประเภทคลาสเรียนใหม่ ๆ นอกเหนือจาก 1 on 1 และ Group Class ในเมนูตั้งค่าได้ (เช่น Yoga, Pilates, Zumba) พร้อมกำหนดความจุเริ่มต้น (Default Capacity)
  - เมื่อสร้างแล้ว ระบบจะนำชื่อประเภทคลาสใหม่ไปเพิ่มเป็นตัวเลือกในฟอร์มการสร้างคลาส และสร้างปุ่มตัวกรองในฝั่งนักเรียนให้สลับดูและจองแบบไดนามิกทันที

- **ระบบจัดการคลาสเรียน**: ฟังก์ชันเปิดคลาสสิทธิ์แอดมิน/เทรนเนอร์, แก้ไขข้อมูลคลาสผ่าน Modal, และฟังก์ชันยกเลิก/ลบคลาสที่คืนสิทธิ์เครดิตให้อัตโนมัติ
- **ระบบตัวกรองแบบไดนามิก**: ตัวเลือกการเปิดคลาสในฟอร์ม และตัวกรองคลาสเรียนของนักเรียนจะเรนเดอร์ตามประเภทคลาสที่มีจริงในฐานข้อมูลอัตโนมัติ

## 🧪 การตรวจสอบการทำงาน (Verification Plan)

### 1. ทดสอบการสมัครสมาชิกและการล็อกอิน
- สมัครสมาชิกใหม่ ตรวจสอบความถูกต้องของการจัดเก็บข้อมูลใน `localStorage`
- ล็อกอินด้วยสิทธิ์นักเรียน ตรวจสอบว่ามีหน้าปฏิทินและเครดิตขึ้นแสดงผลถูกต้องในทันที

### 2. ทดสอบการเพิ่มคลาสเรียนประเภทใหม่และการกรองข้อมูล
- ล็อกอินด้วยสิทธิ์แอดมิน ไปที่หน้า Settings -> Manage Class Types เพิ่มประเภทคลาสเรียนชื่อ "Yoga"
- ไปที่ All Sessions สร้างคลาสเรียน "Yoga" และจองให้กับนักเรียน
- ล็อกอินฝั่งนักเรียน ตรวจสอบว่ามีแถบตัวกรอง "Yoga" ปรากฏขึ้น และเมื่อกดกรองระบบแสดงเฉพาะคลาสเรียนโยคะถูกต้อง

### 3. ทดสอบการคืนเครดิตเมื่อแอดมินลบคลาสเรียน
- นำบัญชีแอดมินลบคลาสเรียนที่นักเรียนจองไว้สำเร็จ
- ตรวจสอบยอดเครดิตของนักเรียนคนดังกล่าวว่าได้รับการคืนกลับเข้ามาในบัญชีแพ็กเกจครบถ้วนถูกต้อง
