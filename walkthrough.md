# บันทึกการดำเนินงาน (Walkthrough)

เราได้ดำเนินการอัปเดตระบบของเว็บไซต์จองคลาสเรียน (PT Booking Platform) สำหรับ **LJD Training** ครบถ้วนตามความต้องการของคุณแล้ว โดยมุ่งเน้นความสวยงามระดับพรีเมียม (Premium Dark Glassmorphism) และฟังก์ชันการใช้งานที่สมบูรณ์

---

## 🛠️ รายการงานที่ดำเนินการเสร็จสิ้น (Completed Tasks)

### 1. ระบบยืนยันตัวตนและการเข้าใช้งาน (Authentication Flow)
- **หน้าจอ Login / Register**: สร้างฟอร์มแบบ overlay สวยงามคลุมหน้าเว็บก่อนเข้าสู่ระบบ โดยผู้ใช้ทุกคนจำเป็นต้องลงทะเบียนหรือเข้าสู่ระบบเพื่อใช้งาน
- **สิทธิ์การเข้าถึงแบบแบ่งบทบาท (RBAC)**: 
  - นักเรียนสามารถใช้งานเฉพาะหน้าแดชบอร์ดของนักเรียนเท่านั้น (ปุ่มสลับบทบาทในเมนูหัวข้อจะถูกซ่อนไว้)
  - ครูฝึก (Trainer / PT) จะเห็นแดชบอร์ดจัดการตารางสอนของตัวเอง
  - เฉพาะแอดมิน (Admin) เท่านั้นที่จะมีเมนูสลับบทบาท ("View As") แสดงในหัวข้อเพื่อทดสอบระบบบทบาทต่าง ๆ

### 2. ฟังก์ชันปฏิทินรายเดือน (Interactive Monthly Calendar)
- **การแสดงผลปฏิทินแบบ Interactive**: เพิ่มวิดเจ็ตปฏิทินรายเดือนในหน้าของนักเรียน (Student View) และครูฝึก (Trainer View)
- **จุดระบุสถานะบนปฏิทิน (Calendar Dots)**:
  - **นักเรียน**: แสดงจุดสีเขียวเมื่อมีคลาสเรียนที่เปิดสอน (Available Class) และจุดสีน้ำเงินเมื่อเป็นคลาสเรียนที่ตนเองได้จองไว้ (Your Booking)
  - **ครูฝึก**: แสดงจุดสีเขียวเมื่อมีตารางสอนของครูฝึกในวันนั้น
- **การกรองข้อมูลตามวันที่**: เมื่อผู้ใช้คลิกเลือกวันที่ในปฏิทิน ตารางรายชื่อคลาสที่แสดงในแดชบอร์ดจะถูกกรองแสดงเฉพาะของวันนั้นทันที และมีปุ่มสำหรับล้างตัวกรองอย่างชัดเจน
- **การสลับเดือน**: สามารถกดปุ่มลูกศรย้อนกลับ/ถัดไปเพื่อดูปฏิทินเดือนอื่น ๆ ได้อย่างอิสระ

### 3. กฎเกณฑ์คลาสเรียนและการหักเครดิต
- **Group Class (คลาสกลุ่ม)**:
  - นักเรียนต้องใช้สิทธิ์จองด้วย **"Group Class Credits"** ของตัวเอง และระบบจะหัก 1 credit ต่อการจองคลาส
  - **เงื่อนไขจำนวนคนขั้นต่ำ**: หากจำนวนคนในคลาสน้อยกว่าที่แอดมินกำหนด (ค่าเริ่มต้นคือ 2 คน) ระบบจะแสดงสถานะคลาสว่า **"待成班 (Pending)"** เมื่อมีผู้เข้าร่วมตั้งแต่ 2 คนขึ้นไป ระบบจะอัปเดตสถานะเป็น **"已成班 (Confirmed)"** ให้โดยอัตโนมัติ
- **1-on-1 Class (คลาสส่วนตัว)**:
  - นักเรียนต้องใช้สิทธิ์จองด้วย **"1-on-1 Credits"** ของตัวเอง และระบบจะหัก 1 credit ต่อการจองคลาสส่วนตัว

### 4. กฎและขั้นตอนการยกเลิกจองคลาสล่วงหน้า 24 ชั่วโมง
- **การบังคับใช้กฎ 24 ชั่วโมง**: การกดยกเลิกจองคลาสเรียน (ทั้ง Group Class และ 1-on-1 Class) จะต้องทำล่วงหน้าอย่างน้อย 1 วัน (24 ชั่วโมง) ก่อนเวลาเริ่มเรียน หากไม่ถึง 24 ชั่วโมงระบบจะแสดงข้อความแจ้งเตือนข้อผิดพลาดและไม่อนุญาตให้ยกเลิก
- **การคืนเครดิต**: คืนเครดิตให้นักเรียนอัตโนมัติเมื่อยกเลิกคลาสสำเร็จ 
- **การทดสอบความถูกต้อง**: ระบบรองรับการยกเลิกคลาสที่จองไว้ล่วงหน้า 1 วันขึ้นไปได้ปกติ

### 5. แผงควบคุมตั้งค่าราคาและกติกาสำหรับแอดมิน (Admin Settings Console)
- เพิ่มแท็บ **"Settings"** ในแผงควบคุมของแอดมิน ซึ่งสามารถปรับปรุงข้อมูลได้โดยตรงบนแพลตฟอร์ม:
  - **ค่าเรียน Group Class**: กำหนดราคาสำหรับทดลองเรียนครั้งแรก (ค่าเริ่มต้น 300 NT) และราคาครั้งถัดไป (ค่าเริ่มต้น 600 NT)
  - **จำนวนผู้ร่วมคลาสกลุ่มขั้นต่ำ**: สามารถปรับค่าผู้ร่วมคลาสขั้นต่ำในการเปิดห้องได้ (ค่าเริ่มต้น 2 คน)
  - **ค่าเรียน 1 on 1**: กำหนดราคาแพ็คเกจแบบขั้นบันได (1 คลาส 1800 NT, 10 คลาส 1450 NT/คลาส, 20 คลาส 1350 NT/คลาส)
- ค่าเรียนที่ตั้งค่าจะถูกนำไปอัปเดตแสดงและใช้ในการคำนวณในแดชบอร์ดนักเรียนโดยตรงแบบเรียลไทม์

### 6. ครูฝึกกำหนดตารางสอนได้อย่างอิสระหลายคลาสต่อวัน
- ครูฝึกสามารถกดสร้างตารางสอนคลาสเรียนด้วยตนเองได้ไม่จำกัดจำนวนต่อวัน โดยใช้เครื่องมือเลือกเวลาเริ่มต้นและสิ้นสุดได้อย่างอิสระ
- **ระบบป้องกันเวลาทับซ้อน (Overlap Prevention)**: ระบบจะดักตรวจจับตารางสอน หากครูฝึกคนนั้นมีคลาสสอนในช่วงเวลาคาบเกี่ยวกันในวันนั้นแล้ว ระบบจะแจ้งเตือนและป้องกันไม่ให้เกิดตารางที่ทับซ้อนกัน

### 7. ปรับสีตัวระบุปฏิทินและการแสดงผลทันทีเมื่อเข้าสู่ระบบ
- **ปรับสีปฏิทิน (Calendar Color Swap)**:
  - **Available Class**: ปรับเปลี่ยนสีจุดกลมและแถบด้านล่างของวันเป็น **สีเขียว** (เพื่อความชัดเจนสำหรับคลาสที่เปิดจองได้)
  - **Your Booking**: ปรับเปลี่ยนสีจุดกลมและแถบด้านล่างของวันเป็น **สีน้ำเงิน** (สำหรับระบุคลาสที่จองไว้)
- **แสดงผลปฏิทินทันทีหลังล็อกอิน (Instant Calendar Rendering on Login)**:
  - แก้ไขปัญหา JavaScript Error (`TypeError: Cannot set properties of null`) ในฟังก์ชันภาษา (`applyLanguage`) ที่ขัดขวางการเรนเดอร์ปฏิทินในตอนเริ่มต้น
  - ปรับปรุงการโหลดโค้ดให้หลีกเลี่ยงการแคชไฟล์ (Cache Busting) ด้วยการกำหนดเวอร์ชันของสคริปต์ใน `index.html` ส่งผลให้เมื่อนักเรียนกดล็อกอินเข้ามา ข้อมูลประวัติการจอง เครดิต และหน้าตารางปฏิทินรายเดือนจะอัปเดตและแสดงผลขึ้นมาในทันทีโดยไม่ต้องทำการกดข้ามเดือนหรือรีโหลดซ้ำ

### 8. ระบบจัดการคลาสและเพิ่มประเภทคลาสใหม่สำหรับแอดมิน (Admin Session & Custom Class Type Management)
- **เพิ่มขีดความสามารถการจัดตารางสำหรับแอดมิน (Admin Schedule Control)**:
  - เพิ่มปุ่ม **"Add Class Session"** ในส่วน "All Sessions" เพื่อให้แอดมินสามารถเปิดคลาสเรียนใหม่ ๆ ได้เหมือนกับครูฝึก
  - เพิ่มปุ่ม **"Edit Class"** (รูปปากกา) และ **"Delete Class"** (รูปถังขยะ) บนตารางแสดงคลาสเรียนทั้งหมดในแดชบอร์ดแอดมิน
  - ระบบลบคลาสเรียนจะทำการตรวจสอบและคืนสิทธิ์เครดิต 1-on-1 ให้กับนักเรียนที่ได้จองคลาสนั้น ๆ คืนไปโดยอัตโนมัติเพื่อป้องกันข้อมูลตกค้าง
- **ระบบปรับแต่งประเภทคลาสเรียนใหม่ ๆ (Custom Class Types Settings)**:
  - เพิ่มแผงจัดการ **"Manage Class Types"** ในเมนูตั้งค่าของแอดมินเพื่อรองรับประเภทคลาสเพิ่มเติมที่จะมีขึ้นในอนาคต (เช่น Yoga, Pilates, Zumba)
  - แอดมินสามารถป้อนชื่อประเภทคลาสและระบุจำนวนความจุเริ่มต้น (Default Capacity) จากนั้นกดบันทึกเข้าสู่ระบบได้ทันที
  - ประเภทคลาสที่สร้างขึ้นใหม่จะเข้าไปเพิ่มเป็นตัวเลือกในตัวเลือกการเปิดคลาส (Class Type Select) ในปุ่ม Add/Edit Session ทันที
  - **ตัวกรองแบบไดนามิก (Dynamic Filters)**: ฝั่งของนักเรียน (Student View) ตัวกรองจะสลับมาดึงข้อมูลตามประเภทคลาสที่มีจริงในระบบอัตโนมัติ ส่งผลให้นักเรียนกดกรองดูตามประเภทคลาสใหม่ ๆ เช่น Yoga หรือ Pilates ได้ทันที

### 9. ระบบจำกัดการมองเห็นคลาสสำหรับนักเรียน (Student Class Visibility Constraints)
- **จำกัดการมองเห็นเฉพาะคลาสเรียนของตนเองและคลาสที่เปิดรับสมัครอยู่**: ปรับปรุงหน้าแสดงคลาสฝั่งนักเรียน (Student View) โดยกรองรายชื่อคลาสในตาราง Available Sessions ให้เห็นเฉพาะคลาสที่ตนเองจองไว้ (Booked) หรือคลาสที่ยังมีที่ว่างเปิดรับจองอยู่ (Available) เท่านั้น
- **ซ่อนคลาสที่เต็มแล้ว (Hide Full Sessions)**: หากคลาสใดมีจำนวนนักเรียนจองเต็มความจุ (Full) และนักเรียนคนปัจจุบันไม่ได้จองคลาสนั้น คลาสดังกล่าวจะถูกซ่อนจากมุมมองของนักเรียนทันทีเพื่อป้องกันความสับสน

### 10. แก้ไขปัญหาระบบจัดเก็บข้อมูลคลาวด์และการยืนยันตัวตน (Google Sheets Sync & Auth Fix)
- **ปรับแต่งการทำงานของ Apps Script**: พัฒนาโครงสร้างการเขียน/อ่านข้อมูลใน Google Apps Script ใหม่ให้ทำงานแบบไดนามิก ดึงและส่งข้อมูลด้วยการจับคู่กับหัวคอลัมน์ (Headers) ของ Google Sheets แทนแบบลำดับดัชนี ช่วยป้องกันปัญหากลุ่มคอลัมน์สลับกัน
- **รองรับข้อผิดพลาดช่องว่าง (Whitespace Trim)**: เพิ่มฟังก์ชันตัดเว้นวรรค (`trim()`) บนหัวตาราง ป้องกันปัญหาคีย์มีเว้นวรรค เช่น `"Phone "` หรือ `" Date"` ทำให้ระบบฝั่งเว็บไม่ยอมทำงาน
- **ปรับปรุงเสถียรภาพฝั่งเว็บ**: เพิ่มตัวดักจับความถูกต้องของอาร์เรย์ (`Array.isArray()`) ในส่วนการเรียกตารางข้อมูลจาก LocalStorage ทำให้หากมีข้อมูลที่เคยผิดพลาดตกค้างอยู่ในเครื่อง เว็บจะไม่อาการค้าง (Sync Crash) แต่จะดึงข้อมูลใหม่มาสวมทับทันที
- **ป้องกันปัญหาการแย่งกันเขียนข้อมูลตอน Reset (Race Condition Fix)**: ปรับปรุงฟังก์ชัน `db.init(true)` ของปุ่ม Reset Database ให้ปิดการซิงก์คลาวด์ชั่วคราวก่อนเริ่มเขียนชุดข้อมูล เพื่อป้องกันการส่งคำขอ POST ไปหลังบ้านแบบคู่ขนานพร้อมกันหลายตารางซึ่งจะทำให้ข้อมูลเก่าตีกลับมาทับข้อมูลใหม่ โดยตอนนี้ระบบจะส่งชุดข้อมูลเริ่มต้นที่ถูกต้องทั้งหมดไปบันทึกพร้อมกันเพียงครั้งเดียว
- **ระบบแสดงผลข้อผิดพลาดล็อกอิน (Inline Login Validation Error)**: เพิ่มการตรวจสอบและแสดงผลข้อความความผิดพลาดในการล็อกอินลงใต้ปุ่ม Sign In โดยใช้ `.error-message` (สีแดง) เพื่อให้ผู้ใช้รับรู้ปัญหาได้ทันที เช่น เมื่อไม่กรอกช่องข้อมูลหรือรหัสผ่านผิด
- **แก้ปัญหาสมัครและล็อกอินผ่านอีเมลล้มเหลว (Google Sheets Email-Username Mismatch Fix)**: เพิ่มการจับคู่ (Mapping) ฟิลด์ `Username` จาก Google Sheets ให้เข้ากับฟิลด์ `Email` ของเว็บไซต์แบบอัตโนมัติทั้งตอนเขียนและอ่าน เพื่อแก้ปัญหาล็อกอินด้วยอีเมลแล้วระบบหาผู้ใช้ไม่พบเนื่องจากหัวตารางใน Google Sheets เป็น Username และมีค่าว่างอยู่ โดยระบบจะดึงค่าเริ่มต้นบัญชีผู้ใช้ระบบ (Default Credentials) มาเติมให้อัตโนมัติในตอนโหลดครั้งแรกและอัปเดตกลับไปยังคลาวด์ทันที

## 📸 การตรวจสอบการทำงาน (Verification Results)

หน้าตาแอปพลิเคชันเวอร์ชันใหม่ประกอบด้วยฟอร์มยืนยันตัวตน, ปฏิทินกรองคลาสเรียน, และการอัปเดตกติการาคาคลาสจองได้รับการพัฒนาเรียบร้อยแล้วและพร้อมเปิดเข้าใช้งานได้ด้วยการรันสคริปต์ `start.sh` หรือ ดับเบิ้ลคลิกสคริปต์ `start.command` ครับ
